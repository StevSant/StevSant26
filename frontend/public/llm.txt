## Architecture

The app is split into a public-facing portfolio and an authenticated admin dashboard, both served from a single Angular frontend backed by Supabase.

### Project Structure

```
frontend/src/app/
  core/             Singleton services, models, guards
    models/
      base/         BaseEntity, PolymorphicEntity, Language, getTranslation util
      entities/     15 domain models (profile, skill, project, experience, etc.)
      form-data/    13 form DTOs (*-form-data.model.ts)
    services/       21 injectable services
    guards/         Route guards (auth, no-auth)
  features/
    auth/           Login page
    dashboard/      Admin CRUD for all entities + analytics (10+ sections)
    portfolio/      Public-facing portfolio pages (14 routes)
  shared/
    components/     29 reusable components (forms, uploads, dialogs, skeletons, etc.)
    config/         App constants (analytics, file-size, geocoding, message-duration)
    directives/     countUp, scrollReveal
    pipes/          translate, markdown, safeHtml
    utils/          flag-utils, skill-icons, download-file

supabase/
  tables/           SQL table definitions (current schema)
  migrations/       21 incremental SQL migrations
  functions/        17 SQL functions (analytics, admin)
  rls/              Row-Level Security policies
  seed/             Demo/seed data
```

### Path Aliases

- `@core/*` -> `src/app/core/*`
- `@shared/*` -> `src/app/shared/*`
- `@features/*` -> `src/app/features/*`

## Key Patterns

### Standalone Components Only
No NgModules. All components are standalone with route-level lazy loading.

### Signal-Based State
Uses Angular signals for reactivity instead of heavy RxJS patterns.

### Custom i18n System
Not ngx-translate. A custom signal-based `TranslateService` with dot-notation keys. Translation files at `frontend/src/assets/i18n/{es,en}.json`. Default language is Spanish.

### Generic CRUD Service
`CrudService` provides generic Supabase CRUD operations for all data access.

### Entity Model Hierarchy
All entities extend `BaseEntity` (id, created_at, user_id). Translatable entities use separate `*_translation` tables. `PolymorphicEntity` pattern links entities via `skill_usage` (projects, experiences, competitions, events).

### Dashboard CRUD Pattern
Each entity follows a consistent structure: `{entity}-list.ts` (list with filtering/pagination), `{entity}-item.ts` (row item), `{entity}-form.ts` (create/edit), `{entity}-form-base-info.ts` (base info tab), `{entity}-form-translations.ts` (i18n tab).

### Theming
`ThemeService` supports 4 themes: Dark Elegant, Light Elegant, Midnight Blue, Warm Sepia.

### SSR & PWA
Static prerendering via `@angular/build:application` with `outputMode: "static"`. Service worker configured via `ngsw-config.json` for PWA support.
