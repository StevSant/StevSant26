# StevSant26 Portfolio

> Personal portfolio website for Bryan Steven Menoscal Santana.

Live site: https://stevsant.vercel.app
Source: https://github.com/StevSant/StevSant26

## Tech Stack

- Angular 21 (standalone components, signals, SSR with hydration)
- Tailwind CSS 4 + Angular Material 21
- Supabase (PostgreSQL, auth, storage)
- TypeScript 5.9 (strict mode)
- Vitest for testing
- Deployed on Vercel

## Architecture

The app is split into a public-facing portfolio and an authenticated admin dashboard, both served from a single Angular frontend backed by Supabase.

### Project Structure

```
frontend/src/app/
  core/             Singleton services, models, guards
    models/
      base/         BaseEntity, PolymorphicEntity, Language, getTranslation util
      entities/     Domain models (profile, skill, project, experience, etc.)
      form-data/    Form DTOs (*-form-data.model.ts)
    services/       Injectable services (CRUD, auth, translate, storage, etc.)
    guards/         Route guards (auth, no-auth)
  features/
    auth/           Login page
    dashboard/      Admin CRUD for all entities + analytics
    portfolio/      Public-facing portfolio pages
  shared/
    components/     Reusable components (forms, uploads, dialogs, etc.)
    config/         App constants
    directives/
    pipes/
    utils/

supabase/
  tables/           SQL table definitions (current schema)
  migrations/       Incremental SQL migrations
  functions/        SQL functions
  rls/              Row-Level Security policies
  seed/             Demo/seed data
```

### Path Aliases

- `@core/*` → `src/app/core/*`
- `@shared/*` → `src/app/shared/*`
- `@features/*` → `src/app/features/*`

## Key Patterns

### Standalone Components Only
No NgModules. All components are standalone with route-level lazy loading.

### Signal-Based State
Uses Angular signals for reactivity instead of heavy RxJS patterns.

### Custom i18n System
Not ngx-translate. A custom signal-based `TranslateService` with dot-notation keys. Translation files at `frontend/src/assets/i18n/{es,en}.json`. Default language is Spanish.

### Generic CRUD Service
`CrudService` provides generic Supabase CRUD operations for all data access.

### Entity Model Hierarchy
All entities extend `BaseEntity` (id, created_at, user_id). Translatable entities use separate `*_translation` tables. `PolymorphicEntity` pattern links entities via `skill_usage` (projects, experiences, competitions, events).

### Theming
`ThemeService` supports 4 themes: Dark Elegant, Light Elegant, Midnight Blue, Warm Sepia.

## Portfolio Pages

- Home, Skills, Projects, Experience, Education, Events, Competitions, Contact
- Detail pages for projects, experiences, events, and competitions
- Referrer redirect handler

## Dashboard Sections

- Profile, Projects, Skills, Skill Categories, Skill Usages, Experiences, Educations, Events, Competitions, Analytics

## Services

- `AuthService` — authentication flow via Supabase
- `CrudService` — generic Supabase CRUD
- `TranslateService` — custom i18n
- `ThemeService` — theme switching
- `SeoService` — meta tags and SEO
- `JsonLdBuilder` — structured data for search engines
- `StorageService` / `DocumentStorageService` — file uploads to Supabase buckets
- `AnalyticsTrackingService` / `AnalyticsDashboardService` — visitor analytics
- `WebVitalsService` — performance monitoring
- `ToastService` — user notifications
- `LoggerService` — logging
